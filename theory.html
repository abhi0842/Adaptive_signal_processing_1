To study the theory of adaptive filters and their applications in biomedical signal processing

1. Introduction to Adaptive Filters

An adaptive filter is a digital filter whose coefficients are not fixed but are iteratively adjusted by an algorithm that responds to a statistical criterion. Unlike conventional fixed filters, which require prior knowledge of signal statistics, adaptive filters derive their optimal settings directly from observed data. This makes them indispensable in environments where signal characteristics are unknown, time-varying, or non-stationary — all of which are common properties of biological signals.

The fundamental adaptive filter problem is formulated as follows. A primary (observed) signal x(n) contains useful information corrupted by noise or interference. A separate reference signal u(n) is available that is correlated with the interference but uncorrelated with the desired signal. The adaptive filter processes u(n) to produce an output y(n) that approximates the interference component, and subtracts it from x(n) to yield the error signal e(n). The adaptation algorithm drives the filter coefficients toward values that minimise a statistical cost function of e(n), typically the mean-squared error (MSE).

1.1 General Adaptive Filter Structure

The general structure consists of four elements:

  • Input signal u(n): The tap-input vector containing M samples: u(n) = [u(n), u(n−1), …, u(n−M+1)]T
  • Weight vector w(n): The M adaptive coefficients: w(n) = [w0(n), w1(n), …, wM−1(n)]T
  • Filter output y(n): The inner product of weights and input: y(n) = wT(n) u(n)
  • Error signal e(n): The difference between the desired response d(n) and the filter output: e(n) = d(n) − y(n)
y(n) = wT(n) u(n) = w0(n)u(n) + w1(n)u(n−1) + … + wM−1(n)u(n−M+1)
e(n) = d(n) − y(n)
IMPORTANT For real-valued signals (ECG, EEG, EMG), the filter output is always y(n) = wT(n)u(n) using the ordinary transpose. The Hermitian transpose (·)H appears only in the complex-valued generalisation and must NOT be applied to real biosignals.

1.2 Cost Function and Optimisation

The goal of adaptation is to find the weight vector that minimises the expected squared error:

J(w) = E[e2(n)] = E[(d(n) − wTu(n))2]

Expanding this expression and taking the gradient with respect to w gives the Wiener-Hopf equation, whose solution is the optimal Wiener filter:

wopt = Ruu−1 rud

where Ruu = E[u(n)uT(n)] is the M×M input autocorrelation matrix and rud = E[u(n)d(n)] is the M×1 cross-correlation vector. The minimum achievable MSE is:

Jmin = σd2 − rudT Ruu−1 rud

In practice, Ruu and rud are unknown and must be estimated or approximated. LMS and RLS are two practical strategies for approaching the Wiener solution adaptively.

2. The LMS Algorithm

2.1 Derivation from Stochastic Gradient Descent

The LMS algorithm (Widrow and Hoff, 1960) is a stochastic gradient descent method. The true gradient of the MSE surface with respect to w is:

∇J(w) = ∂J/∂w = −2 rud + 2 Ruu w

LMS replaces the statistical expectations with their instantaneous estimates at each time step n, using the single sample (u(n), d(n)):

∇̂J(n) = −2 e(n) u(n)

Substituting into the steepest-descent update rule yields the LMS update equation:

w(n+1) = w(n) + µ e(n) u(n)

where µ is the step size (learning rate or adaptation constant). The factor of 2 has been absorbed into µ, which is the standard convention.

2.2 Complete LMS Algorithm

Initialisation:   w(0) = 0   (zero vector, length M)
Step 1 — y(n) = wT(n) u(n)
Step 2 — e(n) = d(n) − y(n)
Step 3 — Weight update:   w(n+1) = w(n) + μ e(n) u(n)

2.3 Stability Condition and Choice of Step Size µ

The step size µ is the most critical parameter of LMS. For the algorithm to converge in the mean-squared sense, µ must satisfy:

0 < μ < 1 / λmax(Ruu)

Since Ruu is generally unknown in practice, a safe practical bound is:

0 < μ < 1 / (M · Pu)

where M is the filter order and Pu = E[u²(n)] is the average input power. The time constant of convergence for the k-th eigenmode is:

τk = 1 / (4 · μ · λk)
Effect of µ on LMS performance Small µ → slow convergence, low steady-state MSE, stable.   Large µ → fast convergence, high steady-state MSE, unstable if µ > 1/λmax. Optimal µ balances convergence speed against steady-state accuracy.

2.4 Misadjustment

The misadjustment M quantifies the excess MSE above the minimum Wiener filter MSE Jmin due to gradient noise in LMS:

M = (Jss − Jmin) / Jmin ≈ μ · tr(Ruu) = μ · ∑k=1M λk

2.5 Normalised LMS (NLMS)

In environments where input power Pu varies over time (common in biosignals), a fixed µ may violate the stability bound at some instants. The Normalised LMS algorithm solves this by dividing the step size by the instantaneous input power:

w(n+1) = w(n) + [ μ / (ε + ‖u(n)‖²) ] · e(n) · u(n)

where ε > 0 is a small regularisation constant (typically 10−6 to 10−3) that prevents division by zero. For real biosignals with time-varying amplitude (EMG bursts, EEG artefacts), NLMS is strongly preferred.

2.6 Advantages and Limitations of LMS

Property / Limitation Detail
Simple implementation O(M) — only M multiply-accumulate operations per step
Low memory requirement Only the weight vector and input buffer — O(M) storage
Guaranteed convergence Provided μ satisfies the stability bound
Robustness Insensitive to initialisation; naturally recovers from disturbances
Slow convergence Rate depends on eigenvalue spread of Ruu — poor for correlated inputs
Steady-state noise Gradient noise prevents convergence to exact Wiener solution
Fixed μ limitation Single μ cannot simultaneously optimise convergence speed and misadjustment
Sensitive to input power Fixed μ may violate stability if input power changes (use NLMS)

3. The RLS Algorithm

3.1 Cost Function and Derivation

The Recursive Least Squares algorithm minimises a weighted least-squares cost function using all available data up to the current time n:

J(n) = ∑i=1n λn−i · e²(i)

where λ ∈ (0, 1] is the forgetting factor and e(i) is the a priori prediction error:

ξ(n) = d(n) − wT(n−1) u(n)

The exponential weighting λn−i assigns full weight to the most recent sample and progressively de-emphasises older data. For λ = 1, all past data is equally weighted (infinite memory); for λ < 1, the algorithm has finite effective memory.

3.2 The Forgetting Factor λ

The forgetting factor is the most important tuning parameter in RLS. The effective memory length — the number of past samples that significantly influence the current weight estimate — is:

Neff = 1 / (1 − λ)
KEY RESULT — Effective memory examples λ = 0.90 → Neff = 10 samples (tracks very fast changes)  |  λ = 0.95 → Neff = 20  |  λ = 0.98 → Neff = 50  |  λ = 0.99 → Neff = 100  |  λ = 1.00 → Neff = ∞ (standard least squares, no forgetting)

A smaller λ gives shorter memory and faster tracking of non-stationarities, but noisier weight estimates. A larger λ gives more stable estimates but slower tracking. For most practical biosignal applications, λ is chosen in the range 0.95 to 0.99.

3.3 Initialisation

The RLS algorithm requires two initialised quantities:

w(0) = 0   (zero weight vector, length M)
P(0) = δ−1 I

where I is the M×M identity matrix and δ is a small positive regularisation constant. A typical choice is δ ∈ [0.001, 0.01] for normalised input signals.

IMPORTANT The parameter δ controls the magnitude of the initial inverse correlation matrix P(0), not the signal-to-noise ratio. Small δ → large P(0) → high initial gain → fast early adaptation. Large δ → small P(0) → conservative start. δ has no direct relationship to SNR.

3.4 Complete RLS Algorithm

Initialisation:   w(0) = 0,   P(0) = δ−1I   (δ ∈ [0.001, 0.01])
Step 1 — ξ(n) = d(n) − wT(n−1) u(n)
Step 2 — k(n) = [ λ−1 P(n−1) u(n) ] / [ 1 + λ−1 uT(n) P(n−1) u(n) ]
Step 3 — w(n) = w(n−1) + k(n) · ξ(n)
Step 4 — P(n) = λ−1 P(n−1) − λ−1 k(n) uT(n) P(n−1)

The gain vector k(n) plays the same conceptual role as the step size µ in LMS, but it is signal-dependent and time-varying. RLS converges in approximately 2M steps regardless of the eigenvalue spread of Ruu — a major advantage over LMS.

3.5 Relationship to the Kalman Filter

RLS is mathematically equivalent to a Kalman filter applied to a specific state-space model where the parameter vector w is constant (zero process noise), the observation is a linear function of the input, and the measurement noise is white. This is a precise equivalence, not an approximation. The Kalman filter is a more general framework that allows for non-constant (evolving) parameter models by adding a process noise term.

3.6 Advantages and Limitations of RLS

Property / Limitation Detail
Very fast convergence Converges in approximately 2M iterations, regardless of input eigenvalue spread
Optimal (least-squares) Minimises the exact weighted sum of all past squared errors
Excellent non-stationarity tracking Forgetting factor provides controlled finite memory
High accuracy Approaches optimal Wiener solution faster than any gradient method
O(M²) complexity M×M matrix-vector products required at every step — expensive for large M
O(M²) memory Must store and update the full M×M inverse correlation matrix P(n)
Numerical sensitivity P(n) may lose positive-definiteness for small λ over many iterations — use square-root RLS
Choice of λ critical Wrong λ causes slow tracking (large λ) or noisy estimates (small λ)

4. LMS vs. RLS: Comparative Summary

Property LMS RLS
Algorithm type Stochastic gradient descent Deterministic weighted least squares
Cost function Instantaneous squared error e²(n) Exponentially weighted sum of past errors
Tuning parameter Step size μ Forgetting factor λ
Stability condition 0 < μ < 1 / λmax(Ruu) Numerically sensitive for λ << 1
Convergence speed Slow — O(1/μ) iterations Fast — approximately 2M iterations
Complexity per step O(M) — ~3M operations O(M²) — ~4M² operations
Memory requirement O(M) — weight vector only O(M²) — stores M×M matrix P(n)
Tracking non-stationarity Moderate — depends on μ Excellent — controlled by λ
Best for Low-resource, mildly non-stationary environments High-accuracy, rapidly non-stationary environments
